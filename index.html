The Problem

Your reps are using AI.
Your team isn't.

Individual AI use is table stakes at this point. The issue is that it's not transferable. A skill built for someone else's workflow — with their customers, their methodology, their context — tells Claude nothing about yours. Install it and you're still starting from zero. Every session. Every rep. Every new hire.

The gap isn't a better skill. It's the layer underneath: a shared foundation where your ICP, your methodology, your objection playbook, and your competitive intel are baked in once — and every skill, every session, and every team member automatically starts from the same intelligence.

Skills without a system

Still individual, still ephemeral

  • Great skill, zero context about your actual customers
  • Each rep installs the same skill, gets different results
  • No shared ICP, methodology, or competitive intel baked in
  • Someone leaves — their custom setup leaves with them
  • New hire installs the skill, still starts from a blank slate
  • The team is 5 individuals, not one compounding system

Shared infrastructure, compounding intelligence

  • Every skill knows your ICP, methodology, and voice
  • One update to the shared brain — every rep benefits instantly
  • New hire onboards in minutes with full team context
  • Knowledge lives in the repo, not in people's heads
  • System gets smarter from every call and campaign
  • The team compounds — not just the person who set it up

The Architecture

Three layers. One system.
Skills go in layer two.

Skills belong in this architecture — they're an important part of it. But they're one shelf in a three-storey building. The building is what makes them actually work for a team. Each layer has a clear job. Together, they turn Claude from a collection of individual tools into shared company infrastructure.

1

The Machine  —  Your Personal Setup

~/.claude/  ·  Per person, takes 20 minutes to configure

▾

This is what lives on each person's laptop. It's individual, but it follows a team standard. Every team member sets this up the same way from a shared template.

~/.claude/ ├── CLAUDE.md # Personal preferences: language, tone, working style ├── settings.json # Safety hooks — blocks rm -rf, git push --force, DROP TABLE └── shared-skills/ # ← Symlink to The Brain (shared GitHub repo) ├── CLAUDE.md # Team identity, methodology, ICP context ├── ICP.md # Living ideal customer profile └── skills/ # All shared slash commands

The symlink is the key insight. Each person has a local folder that points to the shared GitHub repo. When anyone on the team updates the ICP or adds a new skill, everyone gets it immediately on the next git pull.

Safety hooks matter for teams. Configure a PreToolUse hook that blocks dangerous commands — rm -rf, git push --force, DROP TABLE. One bad prompt can't delete a database or force-push to main. Set this in settings.json and make it part of the onboarding template.
2

The Brain  —  Your Company's Shared Intelligence

GitHub repo  ·  The heart of the whole system

▾

One GitHub repository that the entire team syncs. This is where your company's intelligence lives. When anything in here changes, everyone's Claude changes. Commit, pull, done.

The knowledge files

These are the files that make Claude actually useful for your business. They're not generic — they're built from your real calls, your real wins, your real customer language.

File What it contains How it stays current
ICP.md Ideal customer profile — company type, decision maker, pains, desires, triggers, objections, real quotes Auto Updated monthly from call data
METHODOLOGY.md Your sales framework — how you open, qualify, handle objections, and close Team Updated when methodology evolves
OBJECTIONS.md Every objection you encounter + the best response, written in your voice Team Any rep can add after a tough call
COMPETITORS.md How you're positioned against each competitor, what to say and what to avoid Manual Updated when deals involve competitors
CASE_STUDIES.md Real customer wins with context — company profile, challenge, outcome, quote Manual Added with every closed-won deal

The shared skills

Slash commands that every team member can use. When your best AE figures out a great prospect research approach, it becomes a shared /prospect-research skill. Not a forwarded Slack message — a permanent, versioned team asset.

/call-review
Analyse a call transcript against your sales framework. Scores each dimension with coaching feedback.
/prospect-research
Given a company name or LinkedIn URL, build a full prospect brief — pain points, triggers, talking points.
/email-writer
Write cold and follow-up emails in your voice, anchored to the ICP and methodology.
/objection-handler
Diagnose why a deal is stalling. Generate scripts to re-engage based on which objection is real.
/proposal-writer
Draft a proposal section by section, pulling from case studies and ICP context automatically.
/linkedin-post
Turn call insights and ICP patterns into LinkedIn posts in your voice — grounded in real customer language.

Why GitHub

Version control for
your team's intelligence.

Using GitHub for your shared brain isn't a technical choice — it's a strategic one. It's the difference between your team's knowledge being ephemeral and being institutional.

🔄

Always in sync

One git pull and every team member has the latest ICP, skills, and methodology. No Slack threads saying "hey did you see the updated objection guide?"

📜

Full history

You can see every change to the ICP going back to day one. When did we start targeting Series B companies? When did we add that new objection? It's all there.

👀

Peer review for knowledge

Want to update the positioning against a competitor? Open a PR. The team reviews it, not just one person's opinion. Your knowledge base gets the same rigour as your code.

🔒

Secrets stay secret

API keys, credentials, and sensitive data live in .env files that are gitignored and blocked from Claude via a deny list. The brain is shared; the secrets are not.

🚀

Onboarding in minutes

New team member? Clone the shared brain, run the setup script, symlink. They start their first session with the full context of every call and campaign that came before them.

🤝

Knowledge outlasts people

When your best rep leaves, their prompt knowledge doesn't go with them. It's already in the repo — committed, versioned, and available to everyone who joins next.

Getting Started

One person sets it up.
The whole team benefits.

Here's the honest answer to "do I need a developer?": someone technical-enough needs to do the setup once — maybe you, maybe your RevOps person, maybe a contractor for a day. After that, your reps never touch configuration. They just use it.

What "technical enough" means here: comfortable with GitHub, can follow a README, not afraid of a terminal. You don't need to be able to write code. The setup is mostly creating text files and running a few commands — not building software.
Done once

The foundation — set up by you or RevOps

1

Create your shared brain repo on GitHub

One private GitHub repository for your whole team — call it something like acme-sales-skills. This is where your company's intelligence will live. Creating it takes two minutes.

2

Write your team's context files

Create three files to start: CLAUDE.md (who we are, how we sell, our tone), ICP.md (your ideal customer — pain points, real quotes, buying triggers), and METHODOLOGY.md (your sales framework). These are plain text files. No coding required. This is the most important step — the quality of these files determines everything else.

3

Build your first skill

A skill is a folder with a text file inside that tells Claude exactly how to do a specific task — call review, prospect research, email writing. Start with the one thing your team does most. Write it once. Every team member has it instantly.

4

Set up each person's machine — 15 minutes each

Install Claude Code (free with an Anthropic subscription). Clone the shared brain repo. Point Claude to it. Add safety guardrails so accidental commands can't do damage. Each setup takes about 15 minutes. After that, the rep never needs to touch it again.

Every day

What your reps actually do

1

Open Claude Code, start working

Open a terminal (or the Claude desktop app), navigate to their work folder, type claude. That's it. Claude already knows the ICP, the methodology, the competitor positioning — all of it — because it reads the shared brain automatically.

2

Use slash commands for common tasks

Type /call-review, paste a transcript, get coaching. Type /prospect-research, describe a company, get a brief. The skills your team built show up as commands. No prompting knowledge required — the expertise is already baked in.

3

The team keeps the brain current

When someone handles a new objection brilliantly, they add it to OBJECTIONS.md and push to GitHub. When you win a big deal, you add the case study. When the ICP shifts, you update the file. These aren't big maintenance tasks — they're 5-minute additions. The whole team benefits immediately.

The best way to start: Pick one skill your team needs this week. Build it. Use it. Get comfortable with the pattern. Then add the next one. The teams that try to build everything at once burn out. The ones that add one skill per week compound steadily for months.
